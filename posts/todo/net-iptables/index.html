iptables 四表五链 gpt介绍,随便看下

iptables 是 Linux 系统中用于管理网络数据包过滤和防火墙功能的工具。它允许你在 Linux 内核的网络栈中创建规则，以控制网络流量的传入和传出。iptables 提供了一个规则引擎，用于检查、修改、转发或丢弃数据包，以实现网络安全策略。

在 iptables 中，有一个概念叫做 “表（tables）"，它用于组织和管理规则。每个表包含多个 “链（chains）"，而每个链又包含一系列规则。通过在特定的链上添加规则，可以根据特定的条件来决定如何处理数据包。

常见的 iptables 表包括以下四个：

filter 表：filter 表是默认的表，用于过滤数据包。它包含了三个默认的链：INPUT、FORWARD 和 OUTPUT。通过在这些链上添加规则，可以控制数据包的接受、转发和发送。

nat 表：nat 表用于网络地址转换（Network Address Translation，NAT）。它包含了三个默认的链：PREROUTING、POSTROUTING 和 OUTPUT。通过在这些链上添加规则，可以实现端口转发、源地址转换等网络地址转换功能。

mangle 表：mangle 表用于修改数据包的特定字段。它包含了五个默认的链：PREROUTING、INPUT、FORWARD、OUTPUT 和 POSTROUTING。通过在这些链上添加规则，可以修改数据包的 TTL（Time To Live）、IP 头部等字段。

raw 表：raw 表用于绕过其他表中的连接跟踪机制。它包含了两个默认的链：PREROUTING 和 OUTPUT。通过在这些链上添加规则，可以对数据包进行原始处理，不受连接跟踪的影响。

而 “4 表 5 链” 是一个常见的概念，它指的是默认情况下 iptables 中的四个表和五个链。具体而言，这里的 “4 表” 指的是 filter、nat、mangle 和 raw 四个表，而 “5 链” 指的是各表中的五个默认链：INPUT、FORWARD、OUTPUT、PREROUTING 和 POSTROUTING。

通过在这些表和链上添加规则，可以对网络数据包进行详细的控制和管理，以满足特定的网络安全需求。

iptables

添加filter规则

iptables -A <链名> <规则>

iptables -A INPUT -s <IP 地址> -p tcp --dport 80 -j ACCEPT

这将在 INPUT 链上添加一条规则，允许指定 IP 地址的 TCP 数据包通过端口 80。

默认表是 filter 表

iptables 不指定表名 默认都是filter表

iptables -t nat -A PREROUTING <规则>

在这个命令中，-t nat 参数指定了使用 nat 表

nat 表

nat表: 用于网络地址转换（Network Address Translation，NAT）。它包含了三个默认的链：PREROUTING、POSTROUTING 和 OUTPUT。通过在这些链上添加规则，可以实现端口转发、源地址转换等网络地址转换功能。

nat表添加代理

用nat表 prerouting链 设置规则:访问192.168.1.20的 80 端口，将流量转发到 8080 端口,需要设置ip

iptables -t nat -A PREROUTING -p tcp -d <目标 IP> --dport 80 -j DNAT --to-destination <目标 IP>:8080

nat表实现负载均衡

iptables 用nat 表 prerouting链 设置规则:访问 192.168.1.20 的 80 端口，将流量负载均衡 分别到 8080 ,8081,8082 端口

iptables -t nat -A PREROUTING -p tcp -d 192.168.1.20 --dport 80 -m statistic --mode random --probability 0.333 -j DNAT --to-destination 192.168.1.20:8080
iptables -t nat -A PREROUTING -p tcp -d 192.168.1.20 --dport 80 -m statistic --mode random --probability 0.333 -j DNAT --to-destination 192.168.1.20:8081
iptables -t nat -A PREROUTING -p tcp -d 192.168.1.20 --dport 80 -j DNAT --to-destination 192.168.1.20:8082

这三条规则将实现随机负载均衡，将访问 192.168.1.20 的 80 端口的流量随机转发到三个服务的端口：8080、8081 和 8082。

解释：

前两条规则使用 –mode random –probability 0.333 参数来实现随机负载均衡。每个规则的概率被设置为 0.333，使得流量以大约相等的概率分别转发到 8080 和 8081 端口。 第三条规则没有指定负载均衡策略，它将剩余的流量转发到 8082 端口。 这样，流量将以近似相等的概率被分发到三个服务的端口，从而实现随机负载均衡。

请注意，这里的概率是基于每个规则的比例来设置的，并且根据数据包的随机性进行负载均衡。如果需要更精确的负载均衡算法，可能需要考虑使用专用的负载均衡器或反向代理服务器来实现。

filter限流限速

限流: 限制单位时间内的流量,比如 访问9090端口 一个ip 一秒钟只能访问一次 ,一个子网段一秒钟只能访问10次 限速: 限制单位时间内的速度,比如 访问9090端口 一个ip 一秒钟只能访问1k,一个子网段一秒钟只能访问10k 真实的需要用nginx或者其他网关处理

filter forward链 和 nat prerouting链 应用场景

forward 主要处理主机流量规则,比如 网络接口 用iptables 实现 访问docker 内部容器ip:port,docker 不开启映射情况下

命令1:iptables -t nat -A PREROUTING -P tcp --dport 9090 -i DNAT --to-destination 172.18.0 2:9000
命令2iptables -A FORWARD -i eth0 -o docker0 -j ACCEPT

上面命令1 是 nat表 prerouting链 设置规则:访问9090端口，将流量转发到 172.18.0 2:9000 上面命令2 是 filter表 forward链 设置规则:允许eth0网卡的流量转发到docker0网卡,这样就可以访问docker内部容器了 为什么 需要命令2,因为默认是不允许转发的,需要设置转发规则?不是很明白,需要再看看

自定义链

其实就是把规则封装到一个链里,然后在filter表的INPUT链上调用这个链,这样就可以把规则封装起来了
